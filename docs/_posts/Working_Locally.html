Creating a Repository

  • Open 2. GitHub Desktop and click on the drop-down menu next to the "Current Repository" in the top-left of the app.
  • Select the "Add" button next to the search bar
  • Select "Create New Repository" from the menu that appears.
  • A pop-up window will appear with several fields/options.
  • Add a Name for the repository.
    • Repo names must be unique.
      • Use a descriptive name.
    • Description is optional.
    • Use the Choose button to select an alternative filepath for the repo.
      • By default, 2. GitHub Desktop saves repo's to a GitHub folder inside of your Documents folder.
  • Make sure to:
    • Check "initialize this repository with a README"
    • Select "Python" from the Git Ignore dropdown menu.
    • Select a license (GNU GPLv3 allows others to use your code for any open-source project. It cannot be used in a closed source package.)
  • Click "Create Repository".
    • You should see the main pane of the app change to look like the image below.

Open the Repository with Jupyter Notebook

  • While it is ultimately up to you if you would rather use Jupyter Notebooks or VS Code, we will walk through the Jupyter Notebook instructions.
  • Click on the "Repository" menu on the menu bar.
  • Select Open in Terminal(/GitBash for Windows)
  • You should now have a new terminal open inside of the same folder as your repo.
  • Start Jupyter Notebook by entering jupyter notebook in your terminal.
    • If you followed the full Local Python Installation instructions, you should be able to jnb instead of the entire phrase.
  • Your terminal should then start the server for Jupyter Notebook and a new web browser tab should open.
  • Click on the "New" button on the top-right of the Jupyter Notebook file browser page.
  • You should see Python (dojo-env) as an option in the dropdown menu.
    • Select it and a new blank "Untitled" notebook should open.
  • Click on the word "Untitled" and enter a more appropriate title, like "Sakila-Core"

Commit Changes with 2. GitHub Desktop

  • Hop back to GitHub desktop and notice that the main display has changed. It should be showing that you have created a new file.
  • Commit your work thus far, by entering a message in the Message box in the lower left corner.
  • After entering your commit message, click "Commit to main"
  • Finally, click on "Publish repository" at the top of the window.

As you continue to work in your notebook, you will want to save it and commit the changes to this repository. More commits is better than less! This ensures your most up to date work is saved!

Tips for Moving from Google Colab to Jupyter Notebook

Google Colab was designed to mimic Jupyter Notebooks. While they are different, they fundamentally work the same way. Here are a few tips for moving from Colab to Jupyter.

  • Auto-Complete and Docstrings are NOT automatic with Jupyter Notebooks.
    • Press "Tab" to get a suggested autocompletion.
      • This works for file paths, too!
    • Press "Shift+Tab" when your cursor is inside the parentheses for a function.
      • Press Shift+Tab 4 times in a row to open the documentation in a larger window.
  • Repos, repos, repos!
    • You will pretty much always want to create a new notebook or analysis inside a GitHub repository. This makes it easier to find the notebook again, share it, and keep essential files stored with your notebook.
  • Accessing Files will work very differently.
    • You will no longer be mounting Google Drive to access your files.
      • Instead, you will download files to your computer (preferably in the same repo folder as your notebook).

4. Resource: Terminal Commands

To get started, open up your terminal or GitBash and try the commands below. Note that most of these work well in the Windows command prompt (cmd) as well, but not all of them.


For an interactive game to help learn terminal/gitbash commands, check out: http://web.mit.edu/mprat/Public/web/Terminus/Web/main.html


Windows users are strongly encouraged to use GitBash instead of the Command Prompt. The Command Prompt does not use the same commands. For a table comparison of commands for a Linux-based shell vs windows' command prompt, see this article.


Present working directory 

pwd

Use this command any time you are unsure where you are currently in your file structure. pwd stands for Present Working Directory.


List files 

ls

Use the ls command to see all the directories and files that are in your current directory. To clear things up, a directory is just a synonym for a folder.


List files (long form) 

ls -l

Use the 'ls -l' command to see a long-form, representation of the directories and files inside your current directory.

You will see some cryptic messages on the side. The 'drwx' thing that you see are just file permissions for that particular directory or file. For example, the first set of '-'s that follow the 'd' are the permissions allowed for the user. If it says 'rwx' it means the user is allowed to read, write and execute the file. If it just says 'r--' means that the user can only read the file. The second set of three '-'s represent the permissions for a group. The last three '-'s represent the permissions for people who are not the user and are not in the particular group that is allowed. We will have to be changing some of these permissions when we deploy our applications in the cloud. For now, it's important to know that there are different file permissions which can either be read, write, or execute, and these permissions can vary depending on if you are the user, or in the group specified in the listing.


List all files (long form) 

ls -a

Use the 'ls - a' command to see ALL the files. Linux flavored operating systems hide files that start with a '.' by default. This command will show all the previous files and hidden files if there are any. The files that start with a '.' are often configuration files.


Changing to the current directory 

cd .

From the previous command, we saw that 'ls - a' will list some interesting file names like '.' and '..' The '.' is reserved to represent the current directory that we are in. If you run the 'cd . ' and then run the 'pwd' command, you'll see that we didn't move to a different directory. We told the computer to change our directory to our current directory which means don't go anywhere!


Changing to the parent directory 

cd ..

Like the 'cd . ' command, the '..' is reserved to represent the parent directory of the current folder that we are in. If you run this command and then run the 'pwd' command, you'll see that we navigated up one directory(folder) structure. If you feel like the File Structure is starting to resemble the DOM (Document Object Model), it's because they are very similar.


Changing to the root directory 

cd /

This command will change to the root directory of your computer. From this folder, you can access your application folders.


Changing to any folder 

cd folder_name

The 'cd' command followed by a space and name of the folder you want to access will move you into that directory, if it exists. This example would move you into the 'folder_name' directory.


Making a new folder 

mkdir new_directory_name

This command will make a new directory named 'new_directory_name'. Whatever argument you pass into this command will be the name of your new file. After making the directory, you could run 'cd new_folder_name' to enter into that directory.


Delete an empty directory(folder) 

rm directory_name
//or
rmdir directory_name


Delete a non-empty directory(folder) 

rm -rf directory_name

If you try to delete a folder with contents inside you will get an error! Our computer wants us to say more explicitly to delete this directory and everything inside of it by passing the '-rf' option. Be careful with this command as could it accidentally delete files you didn't intend to delete or delete some config files that you wouldn't want deleted at all!


Creating a new file 

touch README.md

The 'touch' command will create a new, empty file with the name and extension type that you specify. You can make any type of file if you provide the extension type. Run the above commands, and then run 'ls' and you will see your new files in the listing.

Removing a file 

rm README.md

The 'rm' command removes any file with the filename and extension provided.

Renaming/Moving a file 

mv README.md  new_filename.md
mv new_filename.md ../new_filename.md

The 'mv' command has two uses. First, you can rename a file by first providing the file's name and then providing what you want the file to be renamed to. In the first example, index.html would be renamed to new_name_index.html. Second, it can be used to move a file to a different folder. In the second example, new_name_index.html would be moved up one directory with the same name. In either case, think of this command as moving a file - into its current directory with a different name or into a different directory with the same or different name.


Copying a file 

cp README.md README_copy.md

The 'cp' command will copy a file into the current directory with the new name you provided. In the example, say you were in your Desktop directory and you ran the above command. The Desktop directory would now have index.html and copy.html in it.

Copying a directory 

cp -rf directory_name copy_directory_name

Copying a directory is similar to a file but we need to pass the '-rf' option in order to copy all the internal contents of the directory. First provide the name of the directory you want to copy and then the name of the copy.


Opening current directory in your GUI

On Mac or Linux use the "open" command:

open .

The 'open' command will open your current directory in your GUI (in 'cmd' or command prompt, use "start ." instead). Likewise, you could open a file inside your directory by running something like "open file_name".


On Windows, use the explorer command: 

explorer .

The 'open' command will open your current directory in your GUI (in 'cmd' or command prompt, use "start ." instead). Likewise, you could open a file inside your directory by running something like "open file_name".

6. Dealing with Large Files


Dealing with Files >100 MB

GitHub has a single-file-size limit of 100 MB. This means that attempting to commit any file larger than 100 MB will result in a fatal error when attempting to push your work to GitHub.

  • GitHub Desktop warns you if you are about to save a file that is over this limit using the pop-up below:
  • NOTE: If you filter out all of the requested conditions for your movies, and saved them as gzip-compressed csv's, your files should NOT be over the 100 MB limit!
  • However, should you run into this following warning from GitHub desktop about your data files being too big you should follow the instructions below:
    • DO NOT CLICK COMMIT ANYWAY!!!
    • Click Cancel instead. We are going to tell git to ignore those large files. This means that it will not try to save them to your commit, but it will let you keep them in the folder without complaining
  • For EACH file that was too large:
    • Right click on it from the sidebar of GitHub Desktop.
    • Select "Ignore file"
  • You will see the large files' names disappear from the changes sidebar. Instead you will see that a file called .gitignore has appeared. This file is how github desktop knows which files to ignore.





7. Installing New Python Packages



Package Manager: pip

Objectives:

  • What is a package?
  • What is a package manager?
  • How to install new packages

Even though programming languages come with a lot right out of the box, you'll find yourself wanting to use some functionality that is not included in your language's standard library. For example, since Python can be used for more than just web development, web functionality does not come pre-loaded when we install Python. However, other developers have created packages that we can download so we don't have to reinvent the wheel.

Pip is the package manager we'll use in Python to install these third-party packages. (There are other package managers, but for simplicity and fewer installations, we're sticking with pip because it was included when we installed Python.) Rather than having to go search for packages and versions ourselves, package managers are generally able to find the versions we need with just a single command from our end.

Warning: installing new packages can break others!

Python packages often rely on code found in other packages, then the other package is a "dependency". If the dependency package dramatically alters the code that the other package uses, updating the dependency package could break the other package!!!

It is VERY possible that you will break your dojo-env at some point by installing new packages.

You should consider cloning your dojo-env as a new environment for you to test installing new packages. If you wanted to create a new clone called "new-env", you would enter the conda command below:

conda create --name new-env --clone dojo-env

Then, you should switch to your cloned environment before installing the new package. 

conda activate new-env

For more information on managing your environment, see the Managing Environments page on Anaconda's website.

To Install a New Package

Enter the pip install command in your terminal/GitBash. That command will look like this:

pip install {{package}} <= NOTE: substitute {{package}} with the package you are installing

For example, if you wanted to install pandas-profiling (documentation), you would run the following (in your terminal/GitBash):

pip install pandas-profiling

Important Note: ALWAYS CHECK THE INSTALLATION INSTRUCTIONS FOR ANY NEW PACKAGE!!

Usually, python packages have documentation either on a website like read the docs or as their README for the package repository.

You should always check their instructions, in case there are additional steps/considerations.

For example, if we check pandas-profiling Installation instructions, we would see that they have an additional command that we need to run:

pip install -U pandas-profiling[notebook]
jupyter nbextension enable --py widgetsnbextension


Pip will then download the package requested and any packages that it may require. You will see progress bars and a message indicating if it was successful or not.

If You Encounter a Permissions Error

  • First, try adding the --user flag to the end of your command. For example,
pip install pandas-profiling --user
  • If the user flag doesn't fix it:
    • For Windows users:
      • Try right-clicking on GitBash in the Start Menu and "Run as Administrator", then try again.
    • For Mac Users:
      • Try going into System Preferences > Security and Privacy > click on the Privacy tab.
      • Check the FIles and Folders section on the sidebar.
      • If you do not see Terminal listen in the right sidebar (or if it is grayed out), click on the lock icon in the bottom left corner and then enter your usual password for your computer.
      • Then click on the check box next to Terminal to grant access (or click the + button to add Terminal - which is stored in Applications > Utilities)

Package Manager: conda

Just like we can install python packages from PyPi (using pip), we can also install packages from Anaconda. Check the Installation instructions on the documentation page for the package you want to install and see if they have both a conda and a pip installation option.

Generally, if there is a conda install command available, that will be the better option for your dojo-env.

Note: if you ever break your environment, return to the the Instructions for Install Python Locally. Specifically, see the final lesson for "Updating your dojo-env"




8. Show Hidden Files

Many files are used by our local installation that are "hidden files".

Hidden files are folders or files whose names start with a period. By default, these are not viewable in File Explorer/Finder.

The are numerous situations where you may need to view your hidden files, see the OS-specific instructions below on how to show hidden files.

  • Windows Users:
    • Turn on View Hidden Files and Folders in your File Explorer menu:
      • Microsoft Support Article
    • On the same settings page, also deselect "Hide extensions for known file types and click OK."
  • Mac Users:
    • Use this keyboard shortcut from inside finder: Cmd+Shift+.
    • Use it again to hide hidden files.
